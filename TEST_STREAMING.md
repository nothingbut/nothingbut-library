# 流式响应功能测试指南

## 前置条件

1. **启动 Ollama 服务**
   ```bash
   ollama serve
   ```

2. **确保模型已下载**
   ```bash
   ollama pull qwen2.5:7b
   ```

## 测试步骤

### 测试 1: 基础流式对话

1. 启动应用
   ```bash
   bun run tauri:dev
   ```

2. 点击右上角 "🤖 打开AI" 按钮

3. 等待连接（应显示 "在线" 徽章）

4. 输入测试消息：
   - "你好，请用100字介绍一下你自己"
   - "讲一个很长的故事"
   - "详细解释量子力学"

5. **预期结果**:
   - ✅ 立即显示打字指示器
   - ✅ 文字逐个字/词出现
   - ✅ 光标动画 (▊) 闪烁
   - ✅ 自动滚动到最新内容
   - ✅ 完成后光标消失
   - ✅ 消息保存到历史

### 测试 2: 多轮对话

1. 连续发送多条消息：
   ```
   用户: 1+1等于几？
   AI: [响应]
   用户: 那 2+2 呢？
   AI: [响应]
   用户: 总结一下前面的计算
   AI: [响应]
   ```

2. **预期结果**:
   - ✅ 保留对话上下文
   - ✅ AI 能引用前面的内容
   - ✅ 历史消息正确显示

### 测试 3: 并发控制

1. 快速连续发送 4 条消息（不等待响应）

2. **预期结果**:
   - ✅ 前 3 条立即开始响应
   - ✅ 第 4 条等待前面完成
   - ✅ 没有崩溃或卡死
   - ✅ 所有消息最终都完成

### 测试 4: 错误处理

1. **停止 Ollama 服务**
   ```bash
   # 在 Ollama 终端按 Ctrl+C
   ```

2. 尝试发送消息

3. **预期结果**:
   - ✅ 显示错误提示
   - ✅ 状态变为 "服务离线"
   - ✅ 不会崩溃

4. **重启 Ollama**
   ```bash
   ollama serve
   ```

5. 刷新页面，重新测试

### 测试 5: UI 交互

1. **Enter 键发送**
   - 输入消息，按 Enter
   - ✅ 消息发送

2. **Shift+Enter 换行**
   - 输入消息，按 Shift+Enter
   - ✅ 添加新行，不发送

3. **加载中禁用**
   - 发送长消息
   - 在响应期间尝试发送新消息
   - ✅ 输入框和按钮禁用

4. **关闭面板**
   - 在响应期间关闭 AI 面板
   - 重新打开
   - ✅ 历史消息保留
   - ✅ 没有内存泄漏

### 测试 6: 性能测试

1. **测量首字时间**
   - 发送消息，计时到看到第一个字
   - 目标: < 1秒

2. **测量完整响应时间**
   - 短回复 (50字): < 3秒
   - 中等回复 (200字): < 8秒
   - 长回复 (500字): < 20秒

3. **内存使用**
   - 打开浏览器开发工具 → Performance
   - 发送 10 条消息
   - ✅ 内存稳定，无泄漏

## 常见问题

### Q1: 没有流式效果，一次性显示全部内容
**原因**: Ollama 返回格式问题或网络缓冲
**解决**: 检查 Ollama 版本，确保支持流式

### Q2: 光标不闪烁
**原因**: CSS 动画未加载
**解决**: 刷新页面，检查浏览器控制台错误

### Q3: 消息重复发送
**原因**: 事件监听器未清理
**解决**: 确保组件销毁时调用 `cleanupListeners()`

### Q4: 第一次很慢，后续很快
**原因**: 模型首次加载到内存
**解决**: 正常现象，无需处理
