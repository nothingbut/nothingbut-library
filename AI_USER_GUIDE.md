## 简介

NothingBut Library 集成了强大的 AI 功能，帮助您更好地阅读和理解小说内容。

### 核心功能

- **🤖 AI 对话助手**: 与 AI 讨论书籍内容，获取分析和解答
- **📝 智能摘要**: 自动生成章节摘要（短/中/长三种长度）
- **🔍 语义搜索**: 用自然语言描述查找相关章节

### 技术特性

- **本地运行**: 所有 AI 功能在本地执行，数据不上传
- **隐私保护**: 对话历史和索引数据仅存储在本地
- **离线可用**: 下载模型后无需互联网连接
- **实时响应**: 流式输出，立即看到 AI 回复

## 安装和配置

### 系统要求

- **操作系统**: macOS 12+, Windows 10+, Linux
- **内存**: 至少 8GB RAM（推荐 16GB）
- **磁盘**: 至少 10GB 可用空间（存储模型）
- **CPU**: 支持 AVX2 指令集（2015 年后的处理器）

### 1. 安装 Ollama

Ollama 是本地 AI 模型运行平台，所有 AI 功能依赖它。

#### macOS

```bash
# 方式 1: 使用安装脚本
curl -fsSL https://ollama.com/install.sh | sh

# 方式 2: 使用 Homebrew
brew install ollama

# 方式 3: 下载安装包
# 访问 https://ollama.com 下载 .dmg 文件
```

#### Windows

1. 访问 https://ollama.com
2. 下载 Windows 安装包
3. 运行安装程序
4. 安装完成后，Ollama 会自动启动

#### Linux

```bash
curl -fsSL https://ollama.com/install.sh | sh
```

### 2. 下载 AI 模型

打开终端，运行以下命令：

```bash
# 下载对话模型（约 4.7GB，首次下载需要时间）
ollama pull qwen2.5:7b

# 下载嵌入模型（用于语义搜索，约 274MB）
ollama pull nomic-embed-text
```

**下载时间参考**:
- 100 Mbps 网速: ~10 分钟
- 50 Mbps 网速: ~20 分钟
- 10 Mbps 网速: ~1 小时

### 3. 启动 Ollama 服务

#### macOS/Linux

```bash
# 前台运行（关闭终端会停止服务）
ollama serve

# 后台运行（推荐）
nohup ollama serve > /dev/null 2>&1 &
```

#### Windows

Ollama 安装后会自动作为服务运行，无需手动启动。

如需手动启动：
1. 按 Win+R
2. 输入 `services.msc`
3. 找到 "Ollama Service"
4. 点击 "启动"

### 4. 验证安装

打开浏览器，访问：http://localhost:11434

应该看到：`Ollama is running`

## AI 对话助手

### 打开对话面板

1. 点击右上角 **🤖 打开 AI** 按钮
2. 等待连接（应显示 "在线" 徽章）
3. 开始对话

### 基础用法

#### 发送消息

- **Enter 键**: 发送消息
- **Shift+Enter**: 换行

#### 对话类型

**通用对话**:
```
用户: 你好，请介绍一下你自己
AI: 我是专业的小说阅读助手...
```

**书籍讨论**（在阅读页面）:
```
用户: 这本书的主角是谁？
AI: 根据当前阅读的《[书名]》，主角是...
```

**章节分析**（在阅读页面）:
```
用户: 总结一下这一章的主要内容
AI: 本章主要讲述了...
```

### 高级用法

#### 角色分析

```
用户: 分析一下主角的性格特点
AI: 主角展现出以下性格特点：
1. 坚韧不拔：面对困难时...
2. 善良正直：在 XX 章节中...
```

#### 情节推测

```
用户: 根据当前剧情，你觉得接下来会发生什么？
AI: 根据前面的伏笔和人物关系，我推测...
```

#### 梗和典故解释

```
用户: "天地玄黄，宇宙洪荒" 是什么意思？
AI: 这句话出自《千字文》的开头...
```

### 对话技巧

✅ **好的提问**:
- "分析主角的性格变化"
- "这一章有哪些伏笔？"
- "解释一下书中的修炼体系"

❌ **不太好的提问**:
- "好看吗？"（主观，AI 无法判断）
- "剧透后面的情节"（AI 只知道已读内容）
- "推荐其他书"（超出功能范围）

## 智能摘要

### 功能说明

自动为章节生成摘要，帮助快速回顾内容。

### 使用步骤

1. 打开书籍阅读页面
2. 选择要摘要的章节
3. 点击 **"生成摘要"** 按钮（即将添加）
4. 选择摘要长度：
   - **短**：100-200 字，快速浏览
   - **中**：300-500 字，平衡详细度
   - **长**：800-1000 字，详细回顾

### 摘要示例

**短摘要**（150 字）:
```
本章主角李明在山中修炼时，意外发现了一个古老的洞府。
洞府中有一本秘籍和一枚丹药。服用丹药后，李明的修为
突破到筑基期。同时，他察觉到有人在跟踪自己...
```

**中摘要**（400 字）:
```
清晨，李明独自在后山修炼基础剑法。突然，他注意到
山壁上有一处异常的光芒...（更详细的描述）
```

**长摘要**（900 字）:
```
本章以细腻的笔触描写了主角李明的一次重要机遇...
（完整的情节、人物、环境描述）
```

### 摘要管理

- **自动缓存**: 生成的摘要会自动保存，下次查看立即显示
- **手动刷新**: 如果内容有更新，可以重新生成
- **多长度**: 同一章节可以生成不同长度的摘要

## 语义搜索

### 功能说明

用自然语言描述查找相关章节，比传统关键词搜索更智能。

### 首次使用：建立索引

⚠️ **重要**: 语义搜索需要先为书籍建立向量索引。

#### 索引单本书

1. 打开书籍详情页
2. 点击 **"为此书建立索引"** 按钮
3. 等待索引完成（显示进度）
4. 索引完成后即可搜索

**索引时间参考**:
- 10 章节: ~30 秒
- 50 章节: ~2.5 分钟
- 100 章节: ~5 分钟

#### 批量索引

如果有多本书需要索引，可以使用批量索引功能：

```javascript
// 在浏览器控制台运行
const { indexBook } = await import('$lib/services/ai');
await indexBook('/path/to/workspace', 1); // 索引书籍 ID=1
```

### 使用搜索

1. 打开 AI 面板
2. 切换到 **"搜索"** 标签
3. 输入自然语言描述
4. 查看搜索结果

### 搜索示例

#### 示例 1: 查找情节

**查询**: "主角第一次战斗"

**结果**:
```
第 5 章：初露锋芒 (85% 相关)
预览：李明握紧手中的长剑，面对三名山贼...

第 12 章：生死之战 (72% 相关)
预览：这是一场真正的生死搏杀...
```

#### 示例 2: 查找人物

**查询**: "反派首次出场"

**结果**:
```
第 8 章：阴影 (92% 相关)
预览：一个身穿黑袍的人影缓缓走出...
```

#### 示例 3: 查找情感场景

**查询**: "感人的离别场景"

**结果**:
```
第 23 章：永别 (88% 相关)
预览：师父望着远去的背影，眼中泪光闪烁...
```

### 搜索技巧

✅ **有效的查询**:
- 具体的情节描述："主角获得神秘力量"
- 人物相关："女主角初次登场"
- 情感场景："悲伤的分离"
- 战斗场景："激烈的战斗"

❌ **效果不佳的查询**:
- 太短："战斗"（改为"主角第一次战斗"）
- 太泛："好看的章节"（主观，无法搜索）
- 书名/作者："三体"（使用书籍筛选功能）

### 搜索设置

- **书籍范围**: 选择在哪本书中搜索（或全部书籍）
- **结果数量**: 显示多少个结果（默认 10 个）
- **相似度**: 最低相似度要求（默认 60%）

## 最佳实践

### 性能优化

#### 1. 保持 Ollama 运行

首次使用 AI 功能时，Ollama 需要加载模型到内存（约 10 秒）。保持 Ollama 运行可以避免重复加载。

```bash
# macOS/Linux: 设置开机自启
# 将以下行添加到 ~/.bashrc 或 ~/.zshrc
alias start-ollama="nohup ollama serve > /dev/null 2>&1 &"
```

#### 2. 合理使用摘要缓存

生成的摘要会自动缓存，除非内容有更新，否则不要重新生成。

#### 3. 提前建立索引

新导入书籍后，立即建立索引。索引过程可以在后台进行，不影响阅读。

### 隐私和安全

#### 数据存储

- **对话历史**: 存储在本地数据库 `library.db`
- **向量索引**: 存储在本地数据库
- **模型文件**: 存储在 `~/.ollama/models/`

#### 数据清理

```bash
# 清理对话历史
# 在应用中：设置 → 清理 AI 数据

# 清理向量索引
# 删除数据库中的 chapter_embeddings 表

# 卸载 Ollama 和模型
ollama stop
rm -rf ~/.ollama
```

### 成本考虑

所有 AI 功能完全免费：
- ✅ 无 API 费用
- ✅ 无使用限制
- ✅ 无网络流量消耗（离线运行）

唯一成本：
- 磁盘空间（模型约 5GB）
- 电力消耗（GPU/CPU 计算）

## 故障排查

### 问题 1: AI 功能不可用

**症状**: 点击 AI 按钮无响应，或显示 "服务离线"

**原因**:
1. Ollama 服务未启动
2. 模型未下载
3. 端口被占用

**解决方案**:

```bash
# 1. 检查 Ollama 是否运行
curl http://localhost:11434
# 应该返回: Ollama is running

# 2. 如果未运行，启动服务
ollama serve

# 3. 检查模型是否存在
ollama list
# 应该看到 qwen2.5:7b 和 nomic-embed-text

# 4. 如果模型不存在，下载
ollama pull qwen2.5:7b
ollama pull nomic-embed-text

# 5. 检查端口占用
lsof -i :11434  # macOS/Linux
netstat -ano | findstr :11434  # Windows
```

### 问题 2: AI 响应很慢

**症状**: 发送消息后等待很久才有响应

**原因**:
1. 首次使用（模型加载）
2. 系统资源不足
3. 模型文件损坏

**解决方案**:

```bash
# 1. 首次使用需要 10-15 秒加载模型，之后会很快

# 2. 检查系统资源
# macOS: 活动监视器
# Windows: 任务管理器
# Linux: htop

# 3. 如果内存不足（< 8GB 可用），关闭其他应用

# 4. 如果仍然很慢，尝试重新下载模型
ollama rm qwen2.5:7b
ollama pull qwen2.5:7b
```

### 问题 3: 搜索无结果

**症状**: 语义搜索返回空结果

**原因**:
1. 未建立索引
2. 相似度阈值太高
3. 查询与内容不匹配

**解决方案**:

```bash
# 1. 检查是否已索引
# 在浏览器控制台：
const pool = await sqlx.connect('library.db');
const result = await sqlx.query('SELECT COUNT(*) FROM chapter_embeddings');
console.log('已索引章节数:', result);

# 2. 如果未索引，建立索引
const { indexBook } = await import('$lib/services/ai');
await indexBook('/path/to/workspace', 1);

# 3. 降低相似度阈值到 0.5 或 0.6

# 4. 尝试更具体的查询
```

### 问题 4: 摘要内容不准确

**症状**: 生成的摘要与章节内容不符

**原因**:
1. 模型理解偏差
2. 章节内容太长
3. 特殊格式或编码

**解决方案**:

```bash
# 1. 尝试重新生成（点击刷新按钮）

# 2. 检查章节内容是否正常显示

# 3. 如果问题持续，尝试其他模型
ollama pull llama3.1:8b
# 修改配置使用 llama3.1
```

### 问题 5: 索引失败

**症状**: 建立索引时报错或卡住

**原因**:
1. Ollama 服务中断
2. 章节内容为空
3. 磁盘空间不足

**解决方案**:

```bash
# 1. 检查 Ollama 日志
# macOS/Linux:
tail -f ~/.ollama/logs/server.log

# 2. 检查磁盘空间
df -h  # macOS/Linux
# 需要至少 1GB 可用空间

# 3. 手动索引单个章节测试
const { indexChapter } = await import('$lib/services/ai');
await indexChapter('/path/to/workspace', 1);
```

## 常见问题

### Q1: AI 功能需要联网吗？

**A**: 不需要。下载模型后，所有功能都在本地运行。

### Q2: 对话历史会上传到云端吗？

**A**: 不会。所有数据都存储在本地数据库，不会上传。

### Q3: 可以使用其他 AI 模型吗？

**A**: 可以。修改配置文件中的模型名称即可：

```javascript
// 默认: qwen2.5:7b
// 可选: llama3.1:8b, mistral:7b 等
```

### Q4: 为什么不使用 ChatGPT？

**A**:
- **隐私**: 本地运行更安全
- **成本**: 完全免费
- **离线**: 无需网络
- **定制**: 可以微调模型

### Q5: 模型可以删除吗？

**A**: 可以。如果不再使用 AI 功能：

```bash
ollama rm qwen2.5:7b
ollama rm nomic-embed-text
```

### Q6: 支持多语言吗？

**A**:
- **中文**: ✅ 完美支持
- **英文**: ✅ 完美支持
- **其他语言**: ✅ 大部分支持

### Q7: 可以同时打开多个对话吗？

**A**: 可以。每本书、每个章节都可以有独立的对话。

### Q8: 索引需要多久？

**A**:
- 10 章节: ~30 秒
- 100 章节: ~5 分钟
- 1000 章节: ~50 分钟

### Q9: 索引后可以删除书籍吗？

**A**: 可以。删除书籍时会自动清理相关索引。

### Q10: AI 会理解全部书籍内容吗？

**A**: AI 每次处理有上下文限制（约 2000 字）。但通过向量搜索，它可以"记住"所有章节的语义。
