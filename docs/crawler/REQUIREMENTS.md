# 小说元数据爬虫系统 - 需求文档

**项目名称**: Novel Metadata Crawler
**版本**: v1.0
**创建日期**: 2026-03-15
**目标**: 构建自动化小说元数据采集系统，用于预填充本地图书馆应用

---

## 1. 项目背景

### 1.1 业务需求
NothingBut Library 需要提供丰富的小说元数据（作者、简介、分类、封面等），以提升用户导入和管理小说的体验。手动输入元数据效率低下，需要从网络数据源自动获取。

### 1.2 目标用户
- 开发者：用于预填充本地数据库
- 最终用户：在导入小说时获得自动补全的元数据

### 1.3 预期收益
- 用户导入小说时自动匹配元数据
- 提供封面、作者、简介等完整信息
- 减少用户手动输入工作量

---

## 2. 功能需求

### 2.1 核心功能

#### FR-1: 优书网索引爬虫
**描述**: 从 www.youshu.me 爬取小说索引信息

**输入**: 书籍 ID（自增整数，从 1 开始）
**输出**: 书籍元数据 + 封面图片

**数据字段**:
- `book_id` (必需): 唯一标识符
- `title` (必需): 书名
- `author`: 作者
- `description`: 简介
- `tags`: 标签/分类（JSON 数组）
- `cover_url`: 封面图片链接
- `source_site`: 来源网站（如 qidian, zongheng）
- `source_url`: 来源网站详情页链接
- `update_status`: 更新状态（连载中/已完结）

**业务规则**:
- URL 格式: `https://www.youshu.me/book/{book_id}`
- book_id 从 1 开始递增
- 连续 N 个 ID 不存在时停止（N = 50）
- 封面图片保存到 `data/covers/{book_id}.jpg`

**成功标准**:
- 成功率 > 95%
- 爬取速度: 1000 本/小时
- 数据完整性: 关键字段（书名、作者）完整率 > 98%

---

#### FR-2: 来源网站详情爬虫
**描述**: 根据优书网提供的来源链接，爬取原始网站的详细信息

**支持的来源网站**:
- 起点中文网 (qidian.com)
- 纵横中文网 (zongheng.com)
- 其他主流网站（可扩展）

**数据字段**:
- 继承 FR-1 的所有字段
- `word_count`: 字数
- `chapter_count`: 章节数
- `rating`: 评分
- `category`: 主分类（如：玄幻、都市）
- `sub_category`: 子分类

**存储策略**:
- 每个来源站点单独数据库（如 `qidian.db`, `zongheng.db`）
- 通过 `youshu_id` 关联优书网索引

**成功标准**:
- 解析成功率 > 90%
- 支持至少 2 个主流小说网站

---

#### FR-3: 首次全量爬取
**描述**: 初始化时爬取尽可能多的历史数据

**流程**:
1. 从 book_id = 1 开始
2. 顺序请求每个 ID
3. 遇到有效书籍：
   - 保存元数据到数据库
   - 下载封面图片
   - 如果有来源链接，触发 FR-2
4. 遇到无效 ID：计数器 +1
5. 连续失败 50 次后停止

**状态记录**:
- `last_valid_id`: 最后一个有效 ID
- `total_books`: 总计爬取书籍数
- `failed_ids`: 失败的 ID 列表（用于补爬）

**成功标准**:
- 单次运行爬取 > 1000 本
- 支持中断恢复（从 last_valid_id 继续）

---

#### FR-4: 增量更新爬取
**描述**: 定期检查新增书籍

**触发条件**:
- 定时任务（每日凌晨 2:00）
- 手动触发

**流程**:
1. 读取 `last_valid_id`
2. 从 `last_valid_id + 1` 开始爬取
3. 同 FR-3 的逻辑

**成功标准**:
- 自动执行无需人工干预
- 新书检测延迟 < 24 小时

---

#### FR-5: 失败重试和补爬
**描述**: 对失败的 ID 进行补爬

**重试策略**:
- 立即重试: 最多 3 次
- 指数退避: 1s, 2s, 4s
- 最终失败记录到 `failed_ids`

**补爬任务**:
- 读取 `failed_ids` 列表
- 批量重试
- 更新失败列表

**成功标准**:
- 最终失败率 < 5%

---

### 2.2 非功能需求

#### NFR-1: 性能要求
- **爬取速度**: 1000-2000 本/小时
- **并发请求**: 3-5 个
- **响应时间**: 单个请求 < 5 秒
- **数据库查询**: < 100ms

#### NFR-2: 可靠性要求
- **数据一致性**: 数据库事务保证
- **容错能力**: 网络异常自动重试
- **断点续传**: 支持中断恢复
- **日志记录**: 完整的操作日志

#### NFR-3: 合规性要求
- **Robots.txt**: 遵守网站爬虫协议
- **请求频率**: 1-3 秒随机延迟
- **User-Agent**: 使用真实浏览器标识
- **内容限制**: 仅爬取元数据，不爬取正文

#### NFR-4: 可维护性
- **代码结构**: 模块化设计
- **日志系统**: 分级日志（INFO/WARN/ERROR）
- **配置管理**: 外部配置文件
- **错误处理**: 完善的异常处理

#### NFR-5: 可扩展性
- **新站点支持**: 插件化爬虫设计
- **数据字段扩展**: 灵活的数据模型
- **分布式部署**: 支持多机部署（未来）

## 3. 数据需求

### 3.1 数据存储

#### 主索引库 (youshu.db)
```sql
-- 书籍表
books:
  - id: INTEGER PRIMARY KEY
  - title: TEXT NOT NULL
  - author: TEXT
  - description: TEXT
  - tags: TEXT (JSON)
  - cover_path: TEXT
  - source_site: TEXT
  - source_url: TEXT
  - update_status: TEXT
  - crawled_at: DATETIME
  - updated_at: DATETIME

-- 爬取状态表
crawl_status:
  - id: INTEGER PRIMARY KEY AUTOINCREMENT
  - last_valid_id: INTEGER
  - last_crawl_date: DATETIME
  - total_books: INTEGER
  - failed_ids: TEXT (JSON)
```

#### 来源站点库 (qidian.db, zongheng.db)
```sql
book_details:
  - book_id: INTEGER PRIMARY KEY
  - youshu_id: INTEGER (外键)
  - title: TEXT NOT NULL
  - author: TEXT
  - description: TEXT
  - category: TEXT
  - tags: TEXT (JSON)
  - cover_url: TEXT
  - cover_path: TEXT
  - word_count: INTEGER
  - chapter_count: INTEGER
  - status: TEXT
  - rating: REAL
  - crawled_at: DATETIME
```

### 3.2 文件存储
```
data/
├── youshu.db           # 主索引数据库
├── qidian.db           # 起点数据库
├── zongheng.db         # 纵横数据库
└── covers/             # 封面目录
    ├── 1.jpg
    ├── 2.jpg
    └── ...
```

### 3.3 数据量预估
- 初始爬取: 10,000 - 50,000 本
- 增量更新: 100-500 本/天
- 数据库大小: ~100MB（10,000 本）
- 封面图片: ~500MB（10,000 本，每张 50KB）

## 4. 技术约束

### 4.1 技术选型
- **编程语言**: Python 3.8+
- **HTTP 库**: requests
- **HTML 解析**: BeautifulSoup4
- **数据库**: SQLite3
- **定时任务**: schedule 或 APScheduler

### 4.2 运行环境
- **操作系统**: macOS / Linux / Windows
- **Python 版本**: >= 3.8
- **磁盘空间**: >= 2GB
- **网络**: 稳定的互联网连接

### 4.3 限制条件
- 单机运行（不考虑分布式）
- 不使用代理池（初期）
- 同步爬虫（可后续优化为异步）
