## ✅ 已完成的更新

### 1. Spec 文档更新
- 文件: `docs/superpowers/specs/2026-03-11-nothingbut-library-design.md`
- Section 5.3: 从三栏改为两栏布局
- 详细描述了 4 层树结构和 3 种内容状态

### 2. CategoryTree 组件重写
- 4 层树状结构：
  - L1: 📚 全部小说（根节点）
  - L2: 📁 科幻、历史、玄幻（首级分类）
  - L3: 📂 太空歌剧、末世幻想、古代（次级分类）
  - L4: 书籍节点 + 状态图标

- 书籍状态图标：
  - ✓ (绿色) = 已完本
  - ⏳ (橙色) = 连载中
  - ⚠ (红色) = 已断更

- Mock 数据（中文）：
  - 三体、流浪地球（科幻 > 太空歌剧，完本）
  - 全球高武（科幻 > 末世幻想，连载中）
  - 明朝那些事儿（历史 > 古代，完本）
  - 某未完成小说（历史 > 古代，断更）

### 3. 小说模块页面 (/novel) 重构
- 两栏布局：
  - 左栏 (280px)：分类树
  - 右栏 (自适应)：内容区域

- 三种状态：
  1. 未选书籍：显示提示"请从左侧选择一本书"
  2. 选择书籍：上半部分书籍元数据 + 下半部分章节目录
  3. 选择章节：上半部分章节内容 + 下半部分章节目录

## 🔍 故障排查

### 问题：分类树不显示

**检查**:
1. 浏览器控制台是否有错误
2. CategoryTree 组件是否正确加载

### 问题：点击书籍无反应

**检查**:
1. `onSelectBook` 回调是否正确传递
2. 控制台是否有 console.log 输出

### 问题：章节目录为空

**检查**:
1. `chapters` 状态是否正确更新
2. Mock 数据是否正确加载

### 问题：样式显示异常

**检查**:
1. CSS 变量是否正确定义（app.css）
2. Tailwind CSS 是否正确加载
